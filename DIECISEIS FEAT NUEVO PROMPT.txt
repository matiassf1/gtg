DIECISEIS FEAT NUEVO PROMPT
===========================
Reestructuración del flujo de entrada + Página informativa /info
=================================================================

OBJETIVO
--------
Agregar una página intermedia entre la landing splash (/) y el flujo de registro/login.
El nuevo flujo es: / (splash con logo) → /info (página del club) → /registro o /login.
La página /info actúa como la "cara pública" del club GTG, con toda la información
comercial y los accesos directos para restaurantes y socios.

================================================================================
CAMBIO 1 — app/page.tsx (Landing splash)
================================================================================

QUÉ SE HIZO:
Un cambio de una sola línea: el href del link del logo se modificó de "/bienvenida"
a "/info".

ANTES: href="/bienvenida"
DESPUÉS: href="/info"

POR QUÉ:
La página /bienvenida era la selección empresa/cliente, que ahora quedó desplazada por
el nuevo flujo. El usuario quería una página informativa completa (/info) como paso
intermedio, no ir directo a la selección de roles. La landing sigue siendo idéntica
visualmente: logo centrado con animación, fondo con glow radial y el texto "Tocá para
entrar" parpadeando. Solo cambia el destino del click.

CAMBIO 2 — app/info/page.tsx (Página informativa — NUEVA)
================================================================================

QUÉ SE HIZO:
Se creó una página completamente nueva en app/info/page.tsx. Es un componente cliente
("use client") ya que necesita estado para el menú hamburguesa y el hook de animaciones
de scroll. La página contiene seis secciones principales más un header sticky.

─────────────────────────────────────────────
A) HOOK useInView + Componente FadeIn
─────────────────────────────────────────────
Para las animaciones de scroll se implementó un custom hook usando IntersectionObserver
de la Web API. El hook observa cuándo un elemento entra en el viewport y, cuando lo
hace, activa el estado `inView` y desconecta el observer (para que la animación solo
ocurra una vez al entrar, no al salir).

El componente FadeIn envuelve cualquier contenido y aplica:
- opacity-0 + translate-y-8 (invisible y 32px más abajo) mientras no está en vista
- opacity-100 + translate-y-0 (visible, en posición) cuando entra al viewport
- transition-all duration-700 para una transición suave de 700ms
- Acepta un prop `delay` en milisegundos para escalonar las animaciones dentro de grids

─────────────────────────────────────────────
B) HEADER STICKY
─────────────────────────────────────────────
Implementado con `sticky top-0 z-50 backdrop-blur-md` para que quede fijo al hacer
scroll con un efecto de cristal esmerilado (difumina el contenido que pasa por debajo).

Desktop (md+): Logo GTG pequeño a la izquierda + 4 botones en fila a la derecha:
- "Afiliar Restaurante" → /registro/empresa (variant outline con borde/texto verde)
- "Hacerme Socio" → /registro/cliente (variant outline con borde/texto verde)
- "Ingreso Restaurante" → /login?role=empresa (variant solid verde)
- "Ingreso Socio" → /login?role=cliente (variant solid verde)

Móvil (<md): Logo + botón hamburguesa (icono Menu/X que alterna). Al abrirse, muestra
un dropdown debajo del header con los mismos 4 botones en un grid de 2 columnas (2×2).
Cada botón cierra el menú al hacer click (setMenuOpen(false) en onClick).

CAMBIO 3 — app/(auth)/login/page.tsx (Login con role param)
================================================================================

QUÉ SE HIZO:
Se modificó la página de login para leer el query parameter `?role=` de la URL y
mostrar una UI contextual según si el usuario viene como restaurante o como socio.

PATRÓN TÉCNICO — Suspense wrapper:
useSearchParams() en Next.js 14 requiere que el componente esté envuelto en un
<Suspense> para la renderización estática. Se adoptó el patrón recomendado:
- LoginContent: componente interno con toda la lógica y el JSX
- LoginPage: componente que exporta por defecto, solo envuelve LoginContent en Suspense

Esto evita errores de build en producción donde Next.js no puede renderizar
estáticamente páginas que leen searchParams sin Suspense.

CAMBIOS DE UI:

1. Badge de rol:
   Cuando la URL incluye ?role=empresa o ?role=cliente, aparece un pill/badge
   verde debajo del logo con el ícono correspondiente:
   - ?role=empresa → icono UtensilsCrossed + "Ingreso para Restaurantes"
   - ?role=cliente → icono User + "Ingreso para Socios"
   Sin role param → no se muestra el badge

2. Título dinámico:
   - ?role=empresa → "Bienvenido, Restaurante"
   - ?role=cliente → "Bienvenido, Socio"
   - sin param → "Bienvenido" (comportamiento original)

3. Link del logo:
   Ahora apunta a /info en vez de / para que el usuario pueda volver a la
   página informativa si quiere.

4. Link "Soy cliente" → "Soy socio":
   Pequeño cambio de texto para consistencia con la nueva terminología del club.

NOTA: La lógica de autenticación no cambia. El role param es puramente cosmético:
sirve para personalizar el mensaje de bienvenida. El sistema Next-Auth determina
el rol real del usuario desde la base de datos tras el login exitoso.

================================================================================
FLUJO COMPLETO RESULTANTE
================================================================================

1. Usuario entra a /
   → Ve el logo GTG animado centrado sobre fondo oscuro
   → Click en el logo (o cualquier parte) → navega a /info

2. Usuario está en /info
   → Ve la página informativa completa del club con scroll
   → Desde el header puede acceder directamente a:
     · /registro/empresa (Afiliar Restaurante)
     · /registro/cliente (Hacerme Socio)
     · /login?role=empresa (Ingreso Restaurante)
     · /login?role=cliente (Ingreso Socio)
   → También puede navegar a través de los botones CTA en cada sección

3. Usuario llega a /login?role=empresa
   → Ve el badge "Ingreso para Restaurantes" y el título "Bienvenido, Restaurante"
   → Completa el formulario normalmente
   → Next-Auth autentica y redirige a /empresa/dashboard

4. Usuario llega a /login?role=cliente
   → Ve el badge "Ingreso para Socios" y el título "Bienvenido, Socio"
   → Completa el formulario normalmente
   → Next-Auth autentica y redirige a /cliente/explorar
